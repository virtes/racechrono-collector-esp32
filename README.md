# racechrono-collector-esp32

ESP32-прошивка под PlatformIO для отправки телеметрии в RaceChrono через BLE
DIY CAN-Bus API.

Сейчас проект читает положение дросселя через `SS49E -> ADS1115 A0 -> ESP32`,
давление тормоза через датчик Bosch `0265005303 -> ADS1115 A1 -> ESP32`,
а напряжение АКБ через делитель напряжения на `ADS1115 A2`.
Дроссель публикуется как CAN PID `258`, давление тормоза публикуется как
CAN PID `257`, напряжение АКБ публикуется как CAN PID `259`. Зеленый LED на
`GPIO5` показывает калибровку и состояние BLE-подключения.

## Быстрый старт

```sh
make build
make upload
make monitor
```

Прошить и сразу открыть serial monitor:

```sh
make run
```

Если порт автоопределился неправильно:

```sh
make devices
make run PORT=/dev/cu.wchusbserial10
```

По умолчанию используется окружение `wemos-d1-mini32`. Другое окружение:

```sh
make run ENV=esp32dev PORT=/dev/cu.usbserial-0001
```

## BLE и RaceChrono

BLE-устройство рекламируется как:

```text
RaceChrono Brake BLE
```

RaceChrono setup:

```text
Settings -> Add other device -> Add other device -> RaceChrono DIY -> Bluetooth LE
```

Каналы в Vehicle profile:

| Канал | CAN PID | Equation | Unit |
| --- | --- | --- | --- |
| Brake pressure | `257` | `bytesToUint(raw, 0, 2) / 100.0` | `bar` |
| Throttle position | `258` | `bytesToUint(raw, 0, 2) / 100.0` | `%` |
| Battery voltage | `259` | `bytesToUint(raw, 0, 2) / 1000.0` | `V` |

## Схема подключения

Этот раздел является источником правды по текущей проводке. Он специально
оформлен таблицами с устойчивыми ID, чтобы его было удобно читать человеку и
обновлять агенту.

### Компоненты

| ID | Компонент | Назначение |
| --- | --- | --- |
| `esp32` | ESP32 | Основная плата, питание 3.3 В, I2C и BLE |
| `adc` | ADS1115 | Внешний ADC для аналоговых сигналов дросселя и давления тормоза |
| `throttle_sensor` | SS49E | Датчик открытия дросселя |
| `brake_pressure_sensor` | Bosch 0265005303 | Датчик давления тормозной системы, питание 5 В |
| `boost_converter` | Повышатель напряжения | Питает датчик давления от АКБ |
| `battery` | АКБ | Источник питания повышателя напряжения |
| `battery_voltage_divider` | Делитель напряжения АКБ | Понижает напряжение АКБ для входа `ADS1115 A2` |
| `status_led` | Зеленый LED | Индикатор калибровки и BLE |
| `status_led_resistor` | Резистор 470 Ом | Ограничивает ток светодиода |

### Соединения

| ID | Откуда | Куда | Сигнал / шина | Примечание |
| --- | --- | --- | --- | --- |
| `led-gpio` | `ESP32 GPIO5` | `470 Ом -> LED анод` | `status_led` | GPIO управляет светодиодом через резистор |
| `led-gnd` | `LED катод` | `GND` | `gnd` | Катод светодиода на общую землю |
| `throttle-signal` | `SS49E OUT` | `ADS1115 A0` | `throttle_adc` | Аналоговый сигнал положения дросселя |
| `ss49e-power` | `ESP32 3V3` | `SS49E VCC` | `3v3` | Питание датчика от 3.3 В |
| `ss49e-gnd` | `SS49E GND` | `GND` | `gnd` | Общая земля |
| `boost-in-plus` | `АКБ +` | `Повышатель IN+` | `battery_plus` | Повышатель подключен к АКБ напрямую |
| `boost-in-minus` | `АКБ -` | `Повышатель IN-` | `gnd` | Минус АКБ должен быть общей землей схемы |
| `brake-sensor-power` | `Повышатель OUT+` | `Bosch 0265005303 pin 3 (+)` | `5v_boost` | Питание датчика давления 5 В |
| `brake-sensor-gnd` | `Повышатель OUT-` | `Bosch 0265005303 pin 1 (-)` | `gnd` | Минус датчика давления на общую землю |
| `brake-sensor-signal` | `Bosch 0265005303 pin 2` | `ADS1115 A1` | `brake_pressure_adc` | 0.5 В = 0 бар, 4.5 В = 250 бар |
| `battery-divider-plus` | `АКБ +` | `Делитель напряжения АКБ верхний вход` | `battery_plus` | Напряжение АКБ до делителя |
| `battery-divider-gnd` | `Делитель напряжения АКБ нижний вход` | `GND` | `gnd` | Нижняя точка делителя на общую землю |
| `battery-divider-signal` | `Делитель напряжения АКБ средняя точка` | `ADS1115 A2` | `battery_voltage_adc` | Исходно замерено: 3.97 В АКБ = 1.88 В на `A2`; после калибровок по показаниям 4.37 В при реальных 4.04 В и 3.82 В при реальных 3.94 В используется коэффициент `2.013564` |
| `ads-power` | `ESP32 3V3` | `ADS1115 VDD` | `3v3` | Питание ADS1115 от 3.3 В |
| `ads-gnd` | `ADS1115 GND` | `GND` | `gnd` | Общая земля |
| `ads-scl` | `ESP32 GPIO22` | `ADS1115 SCL` | `i2c_scl` | I2C clock, в коде `kAdcSclPin = 22` |
| `ads-sda` | `ESP32 GPIO19` | `ADS1115 SDA` | `i2c_sda` | I2C data, в коде `kAdcSdaPin = 19` |
| `ads-addr` | `ADS1115 ADDR` | `GND` | `i2c_addr` | Адрес ADS1115: `0x48` |

### Логические шины

| Шина | Что объединяет |
| --- | --- |
| `gnd` | `ESP32 GND`, `ADS1115 GND`, `SS49E GND`, `АКБ -`, `Повышатель IN-/OUT-`, `Bosch 0265005303 pin 1 (-)`, катод LED |
| `3v3` | `ESP32 3V3`, `ADS1115 VDD`, `SS49E VCC` |
| `5v_boost` | `Повышатель OUT+`, `Bosch 0265005303 pin 3 (+)` |
| `battery_plus` | `АКБ +`, `Повышатель IN+` |
| `battery_voltage_adc` | `Делитель напряжения АКБ средняя точка -> ADS1115 A2` |
| `i2c` | `ESP32 GPIO22 -> ADS1115 SCL`, `ESP32 GPIO19 -> ADS1115 SDA` |

### Важно

- Все `GND` должны быть объединены.
- ADS1115 и SS49E питаются от `3V3`, не от `5V`.
- Датчик давления Bosch `0265005303` питается от выхода повышателя `5V`.
- Сигнал на `ADS1115 A1` не должен превышать допустимый уровень входа ADC.
  Если выход датчика может подниматься выше питания ADS1115, нужен делитель
  напряжения или другой согласователь уровня.
- Напряжение АКБ подключается к `ADS1115 A2` только через делитель. В коде
  используется откалиброванный коэффициент `2.013564`, поэтому расчет такой:
  `battery_volts = A2_volts * 2.013564`.
- LED подключается только через резистор 470 Ом, не напрямую к GPIO.
